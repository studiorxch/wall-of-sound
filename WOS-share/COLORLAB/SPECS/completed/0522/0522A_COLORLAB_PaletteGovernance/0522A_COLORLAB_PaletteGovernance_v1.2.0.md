# Core Philosophy

Colorlab is NOT:
- a disposable palette utility
- a temporary color scratchpad
- a flat collection of editable swatches

Colorlab IS:
# atmospheric archival infrastructure

The archive itself is a primary system.

Governance exists to ensure:
- long-term stability
- provenance preservation
- lineage continuity
- reproducibility
- semantic integrity
- future reprocessability

---

# Foundational Doctrine

CRITICAL:
# raw extraction owns truth

Curated palettes own:
# interpretation

This distinction is foundational.

---

# Truth Model

## Extraction Payloads

Extraction payloads represent:
# immutable visual telemetry

They are:
- deterministic
- reproducible
- preserved permanently
- non-destructive

Extraction payloads must NEVER be mutated.

---

# Curated Palettes

Curated palettes represent:
# editable interpretation layers

They may evolve through:
- cleanup
- editing
- variants
- metadata
- organization
- reinterpretation

without altering source extraction truth.

---

# Governance Principle

Colorlab separates:
# truth
from
# interpretation

Equivalent WOS doctrine:

```txt
2D owns truth
2.5D owns presentation
```

Applied to Colorlab:

```txt
raw extraction owns truth
curated palettes own interpretation
```


---

# Lifecycle Definitions

## SOURCE_CANDIDATES

Immutable extraction payload.

Contains:
- raw extracted candidates
- extraction telemetry
- clustering output
- source provenance
- extraction statistics

SOURCE_CANDIDATES may NEVER:
- mutate
- collapse
- rewrite frequencies
- lose candidate information

---

## WORKING_PALETTE

Editable lineage participant.

Used for:
- cleanup
- experimentation
- structural refinement
- tonal balancing
- atmospheric testing

WORKING_PALETTE is:
# governed editable infrastructure

NOT:
- ephemeral UI state
- temporary autosave buffer
- session-only scratch memory

WORKING_PALETTE may participate in:
- revisions
- branching
- derivation
- lineage continuity

Mutable.

Non-final.
