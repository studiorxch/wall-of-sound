# Image Lab — Moodboard Processing Guide v1.0.0

## Purpose

Image Lab is not just a moodboard. It is a filtering system for turning visual fragments into usable project material.

The goal is to move images through a clear process:

> **Collect → Read → Group → Extract → Refine → Convert → Deploy**

Each image should eventually answer:

> **What can this become inside StudioRich, WOS, merch, music visuals, posters, interfaces, or worldbuilding?**

---

## 1. Collect Visual Fragments

Start by adding images without overthinking them.

Collect anything that has a useful signal:

- Logos
- Posters
- Album covers
- UI screenshots
- Signage
- Symbols
- Type treatments
- Color palettes
- Textures
- Product details
- Film stills
- Broadcast graphics
- Interface panels
- Environmental graphics

At this stage, the image does not need to be fully understood. It only needs to feel potentially useful.

**Question:**

> What made this worth saving?

---

## 2. Capture Usable Data From the Image

Once an image is saved, break it down into observable parts.

Capture:

- Subject matter
- Composition
- Typography
- Color palette
- Symbols or icons
- Materials and texture
- Lighting
- Mood
- Era reference
- Cultural reference
- Format
- Visual hierarchy
- Repetition pattern
- Project relevance

Avoid stopping at “this looks cool.” The useful work is identifying **why** it works.

**Question:**

> What is this image doing visually?

---

Each image should receive practical tags that help it become searchable later.

Use tags across several layers:

- handmade

### Mood Tags

**Question:**

> How would I need to find this later?

---

## 4. Define the Unique Qualities

Write down what makes the image distinct.

Look for specific qualities:

- Strange proportion
- Strong type placement
- Useful color relationship
- Iconic silhouette
- Interesting material surface
- Clear information hierarchy
- Good use of negative space
- Memorable symbol logic
- Strong emotional tone
- Repeatable layout structure

This is where an image stops being random reference and becomes a design clue.

**Question:**

> What is the most usable quality here?

---

## 5. Identify Relationships Between Images

Images become more useful when they connect to other images.

Group images by shared signals:

- Same color logic
- Same typography energy
- Same mood
- Same era
- Same symbol family
- Same layout structure
- Same project application
- Same cultural reference
- Same material or texture
- Same worldbuilding function

This turns scattered images into visual families.

**Question:**

> What does this image belong next to?

---

## 6. Generate Requirements

Translate the visual observations into project requirements.

A requirement is not just inspiration. It is a usable instruction.

Examples:

- Use oversized condensed type.
- Keep layouts modular and grid-based.
- Limit palette to black, gold, and signal red.
- Use monoline symbols only.
- Treat all interface text like public signage.
- Make texture feel printed, scanned, or weathered.
- Use icons that feel like infrastructure, not decoration.
- Build title cards with cinematic negative space.
- Design playlist covers as places, not thumbnails.

**Question:**

> What rule can this image teach us?

---

## 7. Distill Into Refined Idea Categories

After enough images are grouped, name the category.

Examples:

- Broadcast HUD Language
- Subway Signal Typography
- Gold Discipline Poster System
- Soft Analog Nightlife
- Outdoor Warning Glyphs
- Playlist-as-Place Covers
- Infrastructure Luxury
- Martial Arts Title Cards
- City Weather Interface
- Archive Sticker System

A strong category should be reusable across multiple projects.

**Question:**

> What larger visual idea is forming here?

---

## 8. Process the Research

Turn image groups into short research notes.

Each research note should explain:

- What the group contains
- Why it matters
- What visual rules are appearing
- Which StudioRich/WOS projects it connects to
- What can be tested next

This prevents the moodboard from becoming a pile of disconnected references.

**Question:**

> What have these images taught us as a group?

---

## 9. Convert Into Reusable Forms

The final step is converting references into assets, systems, or prompts.

Reusable forms can include:

- Prompt templates
- Poster templates
- Color palettes
- Type rules
- Logo directions
- Icon systems
- Texture packs
- Layout grids
- HUD modules
- Playlist cover systems
- Merch graphic systems
- Motion direction notes
- Brand kit components

This is the transformation point:

> Reference becomes working material.

**Question:**

> What can we build from this?

---

## 10. Connect Back to Projects

Every refined image group should be connected to real StudioRich usage.

Possible destinations:

- StudioRich identity
- WOS interface
- WOS broadcast graphics
- Playlist artwork
- Poster campaigns
- Merch graphics
- Track visuals
- YouTube thumbnails
- Obsidian archive notes
- AI image prompts
- Motion tests
- Website sections
- Installation visuals

**Question:**

> Where does this live in the actual system?

# Quick Processing Checklist

Use this checklist for each image or image group.

```md
## Image Lab Processing Checklist

- [ ] Save the image or reference
- [ ] Identify why it was saved
- [ ] Capture visible design data
- [ ] Add style, theme, mood, and use tags
- [ ] Define its unique qualities
- [ ] Connect it to related images
- [ ] Extract design requirements
- [ ] Place it inside a refined idea category
- [ ] Write a short research note
- [ ] Convert it into a reusable form
- [ ] Connect it to an active StudioRich/WOS project
```

# The Core Process

The Image Lab process can be remembered like this:

```text
1. Save what has signal.
2. Describe what is visible.
3. Tag what it belongs to.
4. Extract what makes it useful.
5. Group it with related images.
6. Turn observations into rules.
7. Name the emerging idea category.
8. Convert the category into usable assets.
9. Connect the asset to real projects.
```

---

# Working Principle

A moodboard is passive when it only stores inspiration.

Image Lab should be active:

> **Every saved image should become data, every group should become a visual language, and every visual language should become something StudioRich can use.**
