6. Deterministic Temporal Behavior
Allowed: deterministic pulse phase, seeded shimmer, class-specific pulse rate, subtle asynchronous variance.

Forbidden: Math.random(), frame-random blinking, synchronized global blinking, urgent emergency-like pulsing, false alarm coding.

Reference formula: pulseValue = 1.0 - pulseDepth + pulseDepth * (0.5 + 0.5 * sin(nowMs * pulseHz * TAU + pulsePhase))

7. Seed Rule
Stable seed order: mmsi ? vesselId ? classKey ? 0. Seed must only influence presentation phase and tiny cluster offsets. Must not imply behavior.

8. Bloom Policy
subtle, atmospheric

smaller close-up, softer farther away

dimmer in dense atmosphere

never urgent

never larger than allowed cap

far lights may bloom slightly larger but dimmer

bloom must not obscure hull topology

Amendment 2: suppressUnderAtmosphere ﾑ Missing Formula
Issue: MaritimeClassLightSignature includes suppressUnderAtmosphere but no guidance on how it affects light rendering.

Fix:

diff
+ ## suppressUnderAtmosphere Behavior
+ 
+ suppressUnderAtmosphere is a multiplier (0.0 ? 1.0) that reduces
+ light alpha and bloom under fog/haze/density pressure:
+ 
+ effectiveAlpha = baseAlpha ? (1.0 - pressure ? suppressUnderAtmosphere)
+ 
+ Higher values = more suppression under atmosphere.
+ Military (0.70) suppresses heavily. Passenger (0.22) retains visibility.
+ 
+ This does NOT affect visibilityClass ﾑ atmosphere suppression is additive.
Amendment 3: clusterSpreadPx ﾑ Missing Units Context
Issue: clusterSpreadPx defined but not clear if pixels at standard zoom or if zoom-dependent.

Fix:

diff
+ ## clusterSpreadPx Units
+ 
+ clusterSpreadPx is defined at reference zoom = 13.0.
+ 
+ At other zoom levels, spread should scale linearly:
+ clusterSpread = clusterSpreadPx ? (zoom / 13.0)
+ 
+ This ensures cluster light grouping remains perceptually consistent
+ across zoom levels without hardcoding pixel values.
Amendment 4: pulseDepth ﾑ Missing Definition
Issue: pulseDepth used in pulse formula but not defined.

Fix:

diff
+ ## pulseDepth Definition
+ 
+ pulseDepth controls the intensity variation of the pulse:
+ - 0.0 = no pulsing (constant intensity)
+ - 1.0 = full intensity range (alpha varies from 0 to baseAlpha)
+ 
+ Typical values:
+ - Cargo/Tanker: low pulseDepth (0.08ﾐ0.12) ﾑ steady industrial glow
+ - Tug/Fishing: higher pulseDepth (0.22ﾐ0.24) ﾑ irregular work light
+ - Ferry: moderate pulseDepth (0.16) ﾑ organized commuter pulse
+ 
+ Pulse formula: intensity = baseAlpha ? (1.0 - pulseDepth + pulseDepth ? pulseModulation)
Amendment 5: allowWakeGlow ﾑ Missing Integration with WakeSignature
Issue: allowWakeGlow in light envelope but no guidance on how wake rendering should consume it.

Fix:

diff
+ ## Wake Glow Integration
+ 
+ When allowWakeGlow is true, the wake renderer MAY add a subtle
+ light bloom to wake segments near the vessel.
+ 
+ This should be subtle (alpha ｲ 0.15) and not create false light sources.
+ 
+ Wake glow is optional and independent of 0526C ActiveWakePolish.
+ If wake glow degrades performance, it may be disabled.
Amendment 6: allowReflectionHint ﾑ Missing Integration
Issue: allowReflectionHint in light envelope but no guidance on what it means.

Fix:

diff
+ ## Reflection Hint Integration
+ 
+ allowReflectionHint is reserved for future water reflection systems.
+ 
+ In v1.0.0, this field exists for forward compatibility but does
+ not affect rendering. Future versions may use it to enable
+ subtle light reflections on water surfaces.
Amendment 7: Fallback NONE ﾑ Missing Rationale for When Used
Issue: getFallbackLightEnvelope(reasonCode) mentioned but no specification of fallback behavior.

Fix:

diff
+ ## Fallback Light Envelope
+ 
+ getFallbackLightEnvelope returns:
+ - If reasonCode === "FALLBACK_INVALID_INPUT": NONE mode, zero alpha
+ - If reasonCode === "VISIBILITY_SUPPRESSED": NONE mode, zero alpha
+ - If reasonCode === "DISTANCE_SUPPRESSED": GHOST mode, very low alpha
+ 
+ Fallback must never crash renderer or produce visible artifacts.
+ Debug logging should indicate fallback usage without spamming console.

Final Statement
v1.0.0 is a canonical maritime light behavior specification for WOS presentation systems. It establishes that "at distance, lights become the vessel" ﾑ far vessels should read as living harbor glints, not uniform map icons. The spec defines class-specific light signatures (cargo: slow heavy amber; ferry: organized commuter pulse; tug: irregular working cluster; etc.), distance-based light collapse from DUAL_NAV/CLUSTER down to POINT/GHOST/NONE, deterministic pulse and shimmer (no Math.random), bloom policy (subtle, never urgent), and strict separation from runtime truth. Light communicates presence; it does not define existence. The spec is ready for REVIEW progression to BUILD.
