2. Stale/Low Confidence Definition
A vessel is stale/low-confidence if ANY:

signalConfidence < 0.40

continuityAlpha < 0.50

lifecycleState === "COASTING"

lifecycleState === "DORMANT"

lifecycleState === "STALE"

This ratio measures continuity burden only ﾑ not lifecycle authority.

4. Global Aggregation Rule
text
globalDensityScore = average(sector density scores)
globalContinuityLoad = average(sector continuity loads)
globalClutterPressure = average(sector clutter pressures)
v1.1.0 intentionally uses deterministic averaging ﾑ no adaptive/cinematic/topology weighting. Future weighting requires separate governance review.

5. Sector Budget Source Defined
maxVessels per sector derived from:

PopulationHierarchy zone budgets (preferred)

Ecological zone densityRange.max (from 0523C)

Fallback: global maxVessels distributed by sector area

Budgets are descriptive scaling references only ﾑ exceeding does NOT authorize culling.

6. Evaluation Optimization
May skip evaluation if no changes in:

vessel count

wake count

tier distribution

Maximum cached-result staleness: 1000ms simulation time. Prevents unnecessary recomputation.

7. Pure Functions with Exhaustive Input Contracts
All inputs marked readonly. Functions are pure, deterministic, and side-effect free.

8. AIS Supremacy Explicit
AISPressure may produce HIGH_AIS_PRESSURE but may never suppress AIS vessels. Preserved from v1.0.0.

9. Camera Interface Explicitly Denied
"Camera systems may NOT use densityClass, continuityLoad, clutterPressure, or reasonCodes for cinematic pacing, hero selection, route steering, target promotion, or focus selection."
