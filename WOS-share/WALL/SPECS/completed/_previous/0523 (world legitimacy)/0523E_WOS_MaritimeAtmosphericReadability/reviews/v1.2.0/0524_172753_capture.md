Critical Fixes (Preserve)
1. Weather/Distance Double-Counting Removed
Problem in v1.1.0: effectiveDistanceMeters = distanceMeters / weatherFactor caused weather to affect both distanceFactor and weatherFactor in the readability score formula, double-counting suppression.

v1.2.0 Solution:

distanceFactor derived from raw distanceMeters only

weatherFactor remains the sole weather suppression path in the canonical formula

Weather-adjusted distance moved to optional continuous-visibility mode

This is mathematically correct and prevents excessive visibility suppression.

2. Blur Hint Uses Raw DistanceFactor
v1.1.0: Blur hint used weather-adjusted effective distance (implicitly, through distanceFactor reference ambiguity).

v1.2.0: Explicit rule:

ts
blurHint = clamp(weatherBlurHint + (1.0 - rawDistanceFactor) * 0.25, 0.0, 1.0);
Prevents hidden weather double-counting in blur computation.

3. Continuous-Visibility Mode Separated
v1.1.0: Optional mode implied but not clearly gated.

v1.2.0:

Must be explicitly enabled by implementation config

Default implementation must NOT use continuous-visibility mode unless visibilityMeters is non-null AND configured

Must not apply discrete weatherFactor as a second suppression path
