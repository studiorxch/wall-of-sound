Executive Summary
This specification defines atmospheric readability as a pure interpretation layer ﾑ it consumes authoritative runtime state and returns readability descriptors, but never mutates vessel truth, wake truth, population tiers, or camera behavior. The core doctrine "Atmosphere may suppress visibility. Atmosphere may never suppress existence." is constitutionally sound.

Strengths (Preserve)
1. Core Doctrine: "Atmosphere may suppress visibility. Atmosphere may never suppress existence."
This single sentence defines the entire authority model. Preserve verbatim.

2. Visibility Class Distinction
FULL ? REDUCED ? SILHOUETTE ? MARKER_ONLY ? LIGHT_ONLY ? ATMOSPHERIC_HIDDEN

Critical: ATMOSPHERIC_HIDDEN does NOT mean dormant, deleted, or evicted. Visual only.

3. Label Readability vs Label Eligibility Separation
text
labelEligibility = PopulationHierarchy
labelReadability = AtmosphericReadability
This prevents atmosphere from overriding authority boundaries.

4. Clean Function Signatures
resolveVesselReadability(), resolveWakeReadability(), resolveLabelReadability() ﾑ all pure, no mutation, no runtime writes.

5. Determinism Requirements
Explicitly forbidden: Date.now(), performance.now(), Math.random(), live DOM reads, renderer buffer reads, camera mutation, runtime mutation.

6. Fog and Night Doctrines
Fog preserves nearby HERO/MID readability and light-only ferry paths. Night shifts toward light trails and ferry lanes rather than making the harbor empty. These preserve harbor rhythm under degraded conditions.

7. Provenance Factor
AIS vessels (1.0) retain higher readability than synthetic vessels (0.72). Supports AIS supremacy and background restraint.

8. Clutter Pressure as Advisory
High clutter pressure may suppress labels and reduce BACKGROUND readability but may NOT delete vessels, change population tiers, or alter AIS confidence.

9. No Camera Authority
Explicitly stated: AtmosphericReadability may not select camera targets, promote vessels, create hero moments, alter camera framing, or request cinematic focus.

Required Amendments (Non-Blocking, Clarifications)
Amendment 1: Visibility Class Thresholds ﾑ Missing Integration with Weather/Time
Issue: resolveVisibilityClass uses only score and timeOfDay (for LIGHT_ONLY vs MARKER_ONLY). Weather state (fog, rain, storm) is not considered in visibility class selection, only in score calculation.

Assessment: This is acceptable because weather affects readabilityScore via weatherFactor, which then affects the score thresholds. The function signature already receives context ﾑ the implementation can read context.weatherState if needed.

No change required. The current design is correct.

Amendment 2: Distance Attenuation ﾑ Missing Integration with Visibility Modes
Issue: Distance attenuation uses fixed thresholds (500m, 1500m, 4000m, 8000m). These may need adjustment based on weather (fog reduces visible range).

Fix: Add clarifying note:

diff
+ ## Distance Attenuation and Weather Interaction
+ 
+ The distance thresholds above assume CLEAR weather.
+ Under FOG, RAIN, or STORM conditions, effective visible distance
+ should be reduced by the weatherFactor:
+ 
+ effectiveMaxDistance = baseMaxDistance ? weatherFactor
+ 
+ Example: In FOG (weatherFactor = 0.38), a vessel at 1500m
+ experiences distance attenuation as if it were at ~570m under CLEAR conditions.
+ 
+ This preserves harbor readability under degraded conditions
+ without creating hard culling thresholds.
Amendment 3: Readability Score Formula ﾑ Missing Provenance Factor
Issue: The formula includes provenanceFactor but Section 16 defines PROVENANCE_READABILITY_FACTOR with values (AIS: 1.0, Synthetic: 0.72). The formula should explicitly include it.

Fix: The formula already includes it. Section 8 shows:

text
readabilityScore =
baseTierReadability
? distanceFactor
? weatherFactor
? timeOfDayFactor
? taxonomyResistanceFactor
? clutterFactor
? provenanceFactor
No change required. The formula is complete.

Amendment 4: Wake Readability ﾑ Missing Age Ratio Integration
Issue: WakeReadabilityInput includes ageRatio but no guidance on how it affects wake readability.

Fix: Add clarification:

diff
+ ## Wake Age Readability
+ 
+ Wake readability degrades with age independent of weather:
+ 
+ wakeReadabilityScore = baseScore ? (1.0 - ageRatio ? 0.6)
+ 
+ Even in CLEAR weather, old wakes near expiration should be
+ visually deemphasized. This prevents "frozen wake" artifacts
+ where expired wakes remain visible.
+ 
+ Age degradation is additive with weather and distance attenuation.
Amendment 5: Clutter Pressure ﾑ Missing Definition of Source
Issue: clutterPressure is defined as number (0.0 ? 1.0) but no guidance on who computes it or from what.

Fix: Add definition:

diff
+ ## Clutter Pressure Source
+ 
+ clutterPressure is provided by PopulationHierarchy or
+ the renderer's density assessment system.
+ 
+ It represents visual density stress and may be derived from:
+ - Vessel count within viewport
+ - Wake density within viewport
+ - Overlap/crowding metrics
+ - Renderer LOP pressure
+ 
+ AtmosphericReadability does not compute clutterPressure.
+ It consumes it as an input.
Amendment 6: ATMOSPHERIC_HIDDEN ﾑ Missing Minimum Visible Distance Rule
Issue: ATMOSPHERIC_HIDDEN could theoretically hide vessels that are extremely close but under severe weather. This may be undesirable.

Fix: Add minimum visibility rule:

diff
+ ## Minimum Visible Distance Protection
+ 
+ Vessels within MINIMUM_VISIBLE_DISTANCE_M (default: 200m)
+ must never return ATMOSPHERIC_HIDDEN regardless of weather or time of day.
+ 
+ At this distance, even in dense fog or heavy rain,
+ some visual indication must remain (LIGHT_ONLY or MARKER_ONLY minimum).
+ 
+ This prevents vessels from "disappearing" while still being
+ perceptually relevant to the viewer.
