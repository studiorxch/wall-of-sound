# 1. CORE ARCHITECTURAL PRINCIPLE

The `MaritimeAtmosphericReadability` component functions exclusively as an **Interpretation Layer component**. It acts as a unidirectional filter: it intercepts authoritative data states from the runtime simulation layers and returns visibility descriptors. It does not possess authority over system truth.

- **Runtime owns existence.**
- **Atmosphere owns readability.**
- **Renderer owns presentation.**

### 1.1 Structural Limitations
The module may advise that an object's clarity is heavily obscured, but it is structurally incapable of determining whether that entity actually exists. High density, heavy weather, or night cycles may suppress visual transparency down to absolute blackness, but they do not cause system culling or modify core database indexes.

---

# 2. RUNTIME AUTHORITY BOUNDARIES

### 2.2 Explicit Ownership
The Atmospheric Readability system maintains total authority over the generation and tracking of:
- Entity visibility scores and environmental clarity values.
- On-screen tag readability and text visibility factor recommendations.
- Presentational fallback flags (`SILHOUETTE`, `LIGHT_ONLY`, `MARKER_ONLY`).
- Readability blur and contrast rendering hints.
- Diagnostic reason codes explaining visual suppression.

### 2.3 Strict Non-Mutation Enforcements
The module is strictly read-only. It **MAY NOT** mutate or pass values to variables managed by the following components:
- System tracking coordinates, speed strings, headings, or lifecycle records inside `AISRuntime`.
- Active prioritization brackets, tier weights, or structural label rules inside `MaritimePopulationHierarchy`.
- Static profiling blueprints or geometric envelope properties inside `MaritimeTaxonomyProfiles`.
- Continuous track generation parameters or decay array lifetimes inside `MaritimeWakeAuthority`.
- Random seed intervals or structural population bounds inside `MaritimeSpawnEcology`.
- Hardware graphic pipelines, context drawing loops, or screen canvas buffers inside `MarineRenderer`.
- Viewport track focal bounds or coordinate target selections inside the active camera systems.
