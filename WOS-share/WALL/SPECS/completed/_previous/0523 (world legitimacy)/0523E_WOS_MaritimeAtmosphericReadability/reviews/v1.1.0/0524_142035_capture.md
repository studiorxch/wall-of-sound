Critical Strengths (Preserve)
1. Core Doctrine Unchanged (Preserved)
"Atmosphere may suppress visibility. Atmosphere may never suppress existence."

This remains the constitutional foundation. Preserve verbatim.

2. readabilityScore Non-Authoritative Clause (New)
"readabilityScore is interpretive only. It may NOT be used as: runtime confidence, AIS confidence, vessel priority, lifecycle state, population tier input, spawn ecology input, camera target weight, pacing trigger, gameplay trigger, wake authority input."

Critical addition č prevents score from becoming hidden authority.

3. Minimum Visible Distance Protection (New)
ts
const MINIMUM_VISIBLE_DISTANCE_M = 200;
Vessels within 200m never return ATMOSPHERIC_HIDDEN. Minimum fallback is MARKER_ONLY or LIGHT_ONLY (night).

Prevents nearby vessels from disappearing in fog or night.

4. Weather-Adjusted Distance Attenuation (New)
ts
effectiveDistanceMeters = distanceMeters / clamp(weatherFactor, 0.25, 1.0);
In fog (weatherFactor = 0.38), a vessel at 1500m experiences attenuation as if at ~570m under clear conditions.

Preserves harbor readability under degraded conditions without hard culling.

5. Wake Age Readability Formula (New)
ts
wakeAgeFactor = clamp(1.0 - clamp(ageRatio, 0.0, 1.0) * 0.6, 0.4, 1.0);
wakeReadabilityScore = intensityRaw ? weatherFactor ? timeOfDayFactor ? clutterFactor ? provenanceFactor ? wakeAgeFactor;
Old wakes near expiration are deemphasized even in clear weather. Prevents "frozen wake" artifacts.

6. clutterFactor Derivation (New)
ts
clutterFactor = clamp(1.0 - clamp(clutterPressure, 0.0, 1.0) * 0.65, 0.35, 1.0);
clutterPressure 0.0 ? 1.0, clutterPressure 1.0 ? 0.35. Linear degradation with floor.

7. Blur and Contrast Hints (New)
Explicit weather tables for WEATHER_BLUR_HINT and WEATHER_CONTRAST_HINT with distance and night modifiers.

Renderer hints only č atmosphere does not control shaders.

8. SYNTHETIC_DEEMPHASIS Reason Code (New)
Emitted whenever provenance === "SYNTHETIC_ECOLOGY" and provenance factor is applied.

Traceable visibility reduction for synthetic vessels.

9. Label Readability Projection Guarantee (New)
resolveLabelReadability(input, context) must be equivalent to resolveVesselReadability(input, context).labelReadable. No separate authority path.

10. 0523D Dependency Fixed
Now references 0523D_WOS_MaritimeWakeAuthority_v1.0.1 with note that this refers to the implemented runtime patch state, not a nonexistent governance version. Registry tracks 0523D as v1.1.0 (PATCH_REQUIRED).
