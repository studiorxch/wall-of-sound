Critical Strengths (Preserve)
1. Anti-Automation Doctrine
"Registry metadata must never become direct execution authority."

Explicit forbidden automation uses:

implementationStatus ? automatic runtime activation

freezeDecision ? automatic deployment

runtimeOwner ? execution permission

dependencyStatus ? runtime feature gating

canonicalStatus ? renderer configuration

Allowed automation: registry linting, validation, missing-field detection, duplicate detection, broken-link detection, dependency warnings.

This prevents the registry from becoming a hidden runtime control plane.

2. Shared Root-Cause Issue Coordination
"Some issues appear in more than one spec but share a single root cause. They must be resolved together. Closing one without closing the other is forbidden."

Example: ISSUE-0523A-001 and ISSUE-0523D-003 both describe the same wakeClass enum mismatch. Coordinated resolution required.

This prevents partial fixes that leave cross-spec inconsistency.

3. Constitutional Dependency Records
0522O, 0522P, 0522Q now have full registry entries:

spec_id, canonical_version, stage, freeze_decision

parentSpecs, downstreamSpecs

lastReviewDate, openIssues

Complete governance visibility for the constitutional layer.

Required Amendments (Non-Blocking, Clarifications)

Issue: Issue status transitions show RESOLVED ? VERIFIED but VERIFIED is not in the RegistryIssue.status enum.

  status:
    | "OPEN"

Amendment 2: Missing resolutionDate Field in Registry Snapshot Examples
Issue: Schema includes resolutionDate but snapshot examples don't populate it (null is fine, but note acceptable).

Fix: No change needed Ń null is acceptable. This is an observation, not a blocker.

Amendment 3: Missing registry_format_version Purpose Statement
Issue: registry_format_version: "1.2.0" is present but no doctrine on when it increments.

Fix: Add rationale:

diff
+ ## Registry Format Versioning
+ 
+ registry_format_version increments when:
+ - Schema changes (new required fields, removed fields, type changes)
+ - Enum values change (new stage, freeze, or status values)
+ - Semantic meaning of existing fields changes
+ 
+ registry_format_version does NOT increment for:
+ - Adding new spec records
+ - Updating implementation status
+ - Resolving issues
+ - Changing freeze decision
+ 
+ Consumers should validate registry_format_version compatibility before parsing.
