Required Amendments (Non-Blocking, Clarifications)
Amendment 1: Missing Resolution Target Version for Issues
Issue: Issues have resolution_version: null but no field for target version where fix will be released.

Fix: Add optional field:

diff
+   targetFixVersion: string | null;  // e.g., "v1.2.2" or "v1.1.1"
Amendment 2: Missing Issue Status Transition Rules
Issue: No guidance on when an issue moves from OPEN ? RESOLVED ? VERIFIED.

Fix: Add transition rules:

diff
+ ## Issue Status Transitions
+ 
+ OPEN ? RESOLVED: When code patch is merged
+ RESOLVED ? VERIFIED: When patch is confirmed in staging/production
+ 
+ Issues marked RESOLVED must include:
+ - resolutionVersion (the spec version containing the fix)
+ - resolutionDate
+ 
+ VERIFIED issues may be closed after confirmation.
Amendment 3: No Escalation Path for Stale Blocking Issues
Issue: Blocking issues could remain open indefinitely with no governance mechanism.

Fix: Add escalation rule:

diff
+ ## Stale Blocking Issue Escalation
+ 
+ If a BLOCKING issue remains OPEN for > 14 days:
+ - Issue severity escalates to ARCHITECTURE_REVIEW
+ - Registry owner notified
+ - Dependent specs may request freeze override with documented justification
+ 
+ After 30 days without resolution:
+ - Affected spec(s) may be marked SUPERSEDED or BLOCKED permanently
+ - Exception requires architecture review approval
