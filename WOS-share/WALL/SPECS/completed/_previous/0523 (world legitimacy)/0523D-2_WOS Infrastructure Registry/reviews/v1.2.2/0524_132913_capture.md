Critical Fixes (Preserve)
1. lastBuildDate Semantics Explicitly Defined
lastBuildDate records: the date of the most recent code implementation return. It is NOT updated by: spec review alone, registry maintenance, freeze-state changes, patch spec creation, or issue lifecycle changes. It updates only when implementation code is returned for review.

Prevents date creep, false implementation freshness, stale-state ambiguity, and governance timeline corruption.

2. RESOLVED vs VERIFIED Semantics
State	Meaning
RESOLVED	Fix has been applied in a spec revision or implementation return
VERIFIED	Architectural review has confirmed the fix is correct and contracts are valid
This distinction mirrors BUILT_UNVERIFIED vs BUILT_VERIFIED for implementation lifecycle tracking.
