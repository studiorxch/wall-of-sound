# 0605G.1_WOS_ArticulatedBusLiveBendUpdate_v1.0.0_BUILD

## PURPOSE

Patch the live update gap discovered after 0605G.

0605G successfully created:

```text
front segment
accordion joint
rear segment
bend metadata
```

but live rendering currently has a limitation:

```text
WSL rebuilds a mesh only on definition change.
articulationBendDeg is applied at rebuild boundaries.
bend does not update continuously between feed/render updates.
```

0605G.1 makes articulated bus bend updates visible during live transit presentation.

This is a small focused patch.

It should not expand into:

```text
new bus physics
new route prediction
new camera behavior
new stop/dwell system
new livery system
```

---

# PROBLEM

Current 0605G result:

```text
ArticulatedBusPresentationPass.observe(actor)
→ computes bendAngleDeg
→ BusVisualFallbackRenderer passes articulation metadata
→ WSL builds articulated segment mesh
```

But:

```text
WorldSpaceVehicleLayer mesh rebuild guard
```

does not treat articulation bend changes as a live transform-affecting update.

So the renderer can keep receiving new articulation values while the visible mesh remains visually stale until another rebuild-triggering property changes.

For close hero shots created by 0605F:

```text
follow articulated bus
→ bendy bus should visibly bend
→ current bend can appear frozen
```

That must be fixed before stop/dwell cues or hero shot presets.

---

# CORE DECISION

0605G.1 is a render-presentation patch.

It does not change how articulation is computed.

It changes how WSL applies updated articulation values.

Canonical authority split:

```text
ArticulatedBusPresentationPass computes articulation.
BusVisualFallbackRenderer forwards articulation payload metadata.
WorldSpaceVehicleLayer applies articulation to existing mesh/segment transforms.
```

No truth mutation.

No reclassification.

No camera changes.

# REQUIRED WSL BEHAVIOR

When an existing articulated bus payload is upserted:

```text
same id
same silhouetteClass
same variant
same lod
new articulationBendDeg
```

WSL must update the existing articulated mesh visuals.

Acceptable strategies:

## Strategy A — Rebuild Guard Patch

Treat articulation-relevant values as definition-changing for articulated buses:

```js
articulationBendDeg
metadata.articulationSimplified
metadata.rearLng
metadata.rearLat
```

If changed beyond tolerance:

```text
rebuild mesh
apply new bend
```

## Strategy B — Per-Upsert Transform Hook

Keep existing mesh and update named subparts:

```text
front
joint
rear
```

by applying local rotations/offsets from new `articulationBendDeg`.

Preferred if low-risk.

## Strategy C — Hybrid

Use transform hook for bend-only changes.

Use rebuild when:

```text
simplified changes
LOD changes
silhouette changes
variant changes
```

Recommended:

```text
Strategy C
```

because regional simplification and LOD changes should remain rebuild-safe, while bend-only changes should be lightweight.

---

# BEND UPDATE TOLERANCE

Avoid tiny rebuild/transform churn.

```js
ARTICULATION_BEND_EPSILON_DEG = 0.5
```

Only apply live update when:

```js
Math.abs(nextBendDeg - previousBendDeg) >= 0.5
```

or when simplification state changes.

---

# MESH PART NAMING

WSL articulated builders should name parts consistently.

Required names or userData keys:

```js
frontSegment
jointSegment
rearSegment
```

Recommended:

```js
mesh.userData.part = "front"
mesh.userData.part = "joint"
mesh.userData.part = "rear"
```

The group should store:

```js
group.userData.articulated = true
group.userData.articulationBendDeg = number
group.userData.articulationSimplified = boolean
```

---

# LIVE TRANSFORM MODEL

For full articulation at low/city:

```text
front segment remains aligned to actor heading
joint segment rotates partially by bend
rear segment rotates by bend
```

Suggested mapping:

```js
joint.rotation.z = bendRad * 0.5
rear.rotation.z = bendRad
```

This is visual language, not physics.

For regional simplified:

```text
joint hidden or collapsed
rear hidden or collapsed
single silhouette remains readable
```

For cruise:

```text
no individual bus rendering
```

`busVisualFallbackRenderer` already passes:

```js
metadata: {
  articulationBendDeg,
  articulationSimplified,
  rearLng,
  rearLat
},

```

0605G.1 may normalize this into a single WSL-internal state:

```js
type ArticulationRenderState = {
  bendDeg: number
  simplified: boolean
  rearLng: number | null
  rearLat: number | null
}
```

Use:

```text
payload.articulationBendDeg
```

first, fallback to:

```text
payload.metadata.articulationBendDeg
```

# EXECUTION FLOW

Normal bus update:

```text
BusVisualFallbackRenderer.renderOnce()
→ ArticulatedBusPresentationPass.observe(actor)
→ payload includes articulationBendDeg
→ WSL.upsertVehicle(payload)
→ WSL detects existing articulated mesh
→ WSL applies live bend transform OR rebuilds if required
→ mesh visually updates
```
