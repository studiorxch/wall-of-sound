# 0605K.1_WOS_CameraShotSelectorUI_v1.0.0_BUILD

## PURPOSE

Expose the new actor camera shot presets in the runtime UI.

0605I created the shot vocabulary.

0605K made those shots terrain-aware.

But right now, testing them requires debug commands.

0605K.1 adds the minimal UI bridge:

```text
CAM dropdown
→ select shot
→ ActorCameraShotPresets.applyShot(id)
```

The goal is to make the important camera views testable without console work.

---

# CURRENT CONTEXT

Current visible CAM dropdown contains:

```text
follow
lead
side
high
hide actor
```

These are useful legacy camera modes.

Do not remove them.

Add a small set of shot presets focused on immediate validation:

```text
windshield
left_window
right_window
rear_window
bus_front_window
street_level
```

---

# CORE DECISION

This is not a new camera system.

This is a UI bridge.

Camera logic remains owned by:

```text
0605F TransitCameraTargeting
0605I ActorCameraShotPresets
0605K TerrainAwareActorCamera
```

0605K.1 only exposes existing controls.

# UI BEHAVIOR

When the user selects a new shot option:

```text
1. prevent crash if ActorCameraShotPresets unavailable
2. call SBE.ActorCameraShotPresets.setShot(id)
3. call SBE.ActorCameraShotPresets.applyShot(id)
4. store current UI selection
5. display selected option
```

If no target exists:

```text
do not crash
keep selection
surface no_target in debug/state only
```

If camera unavailable:

```text
do not crash
keep selection
request stored by shot system
```
