# 1. ARCHITECTURAL & CORE DOCTRINE

The `ProceduralVesselTopology` engine constructs expressive structural layouts for rendering engines using simple, highly recognizable shapes rather than complex textures or high-polygon meshes.

- **2D owns truth. 2.5D owns presentation.**
- **Semantic topology over photorealism:** Groupings are recognized by hull aspect ratios, cargo spacing rhythms, superstructure volumes, and structural markers.
- **Low-resolution recognition beats high-detail realism.**
- **Topology is an interpretive presentation layer:** Blueprints are compiled to generic shape arrays; downstream rendering pipelines handle perspective projection.
- **Topology changes do not modify simulation states:** Layout rules can alter visibility classes or render details, but they cannot adjust tracking updates, speed calculations, or orientation values.

---

# 2. RUNTIME AUTHORITY BOUNDARIES

### 2.1 Explicit Domain Ownership
The module asserts exclusive governance over:
- Structural schemas, configuration profiles, and validation limits for vessel blueprints.
- Default internal vector shapes for all 11 canonical vessel classes plus fallback parameters.
- Level of Detail (LOD) optimization state calculations matching viewport parameters.
- Placement maps for deck items, bridge modules, container stacks, and equipment layouts.
- Generation of generic drawing lists using abstract color classifications (`hull`, `deck`, etc.).
- Anchors for lights, navigation points, custom tags, interface hovers, and wake systems.

### 2.2 Prohibited Modification Scopes
The module has **zero authority** over, and is completely blocked from mutating:
- True headings, positional lists, or velocity vectors maintained within `AISRuntime`.
- Lifespan tracking, dissipation steps, or node data buffers inside `MaritimeWakeAuthority`.
- Target priorities, narrative tagging flags, or tier selections within `MaritimePopulationHierarchy`.
- Base visibility ratings or environmental metrics evaluated inside `VisibilityClassRuntime`.
- Color swatches, opacity profiles, line widths, or font configurations inside `MaritimeStyleRegistry`.
- Global view targets, user selection frames, or interaction focus paths inside camera modules.

    if (!rawClass) return "unknown";
    const clean = rawClass.toLowerCase().trim();
    const validClasses: VesselTopologyClassKey[] = [
      "cargo", "tanker", "ferry", "service", "recreational", 
      "fishing", "passenger", "tug", "military", "industrial", "unknown"
    ];
    return validClasses.includes(clean as VesselTopologyClassKey) ? (clean as VesselTopologyClassKey) : "default";
  }

  public getTopologyBlueprint(classKey: string | null | undefined): VesselTopologyBlueprint {
    const normalized = this.normalizeTopologyClass(classKey);
    const bp = this.blueprints.get(normalized) || this.blueprints.get("default");
    if (!bp) {
      throw new Error(`Critical Topology Isolation Defect: Missing default blueprint fallback.`);
    }
    return bp;
  }

  public resolveTopologyLOD(
    visibilityClass: string | null,
    zoom: number | null,
    populationTier: string | null
  ): VesselTopologyLOD {
    if (visibilityClass === "ATMOSPHERIC_HIDDEN" || visibilityClass === "LIGHT_ONLY") return "LIGHT";
    if (visibilityClass === "MARKER_ONLY") return "MARKER";
    if (visibilityClass === "SILHOUETTE") return "SILHOUETTE";

    const z = zoom !== null ? zoom : 0;

    if (visibilityClass === "REDUCED") {
      return z >= DEFAULT_MIN_TOPOLOGY_ZOOM ? "TOPOLOGY" : "SILHOUETTE";
    }

    if (visibilityClass === "FULL") {
      if (populationTier === "HERO" && z >= DEFAULT_MIN_CLOSE_DETAIL_ZOOM) {
        return "CLOSE_DETAIL";
      }
      return z >= DEFAULT_MIN_TOPOLOGY_ZOOM ? "TOPOLOGY" : "SILHOUETTE";
    }

    return z >= DEFAULT_MIN_TOPOLOGY_ZOOM ? "TOPOLOGY" : "SILHOUETTE";
  }

  public createTopologyInstance(input: VesselTopologyInstanceInput): VesselTopologyInstance {
    const classKey = this.normalizeTopologyClass(input.classKey);
    const blueprint = this.getTopologyBlueprint(classKey);
    const lod = this.resolveTopologyLOD(input.visibilityClass, input.zoom, input.populationTier);

    // Calculate length/width physical-to-presentation scale factors
    const lengthBase = input.lengthMeters ? input.lengthMeters : 40;
    const computedScale = lengthBase / 100; 

    // Filter drawing array elements strictly against Level of Detail access controls
    const activePrimitives = blueprint.primitives.filter(p => {
      const pRank = this.getLODRank(p.visibleFromLOD);
      const targetRank = this.getLODRank(lod);
      return pRank <= targetRank;
    });

    return {
      vesselId: input.vesselId,
      classKey,
      blueprintId: blueprint.blueprintId,
      lod,
      primitives: activePrimitives,
      anchors: [...blueprint.anchors],
      scale: computedScale,
      headingLocked: lod !== "LIGHT" && lod !== "MARKER"
    };
  }

  private getLODRank(lod: VesselTopologyLOD): number {
    const ranks: Record<VesselTopologyLOD, number> = {
      LIGHT: 0,
      MARKER: 1,
      SILHOUETTE: 2,
      TOPOLOGY: 3,
      CLOSE_DETAIL: 4
    };
    return ranks[lod];
  }
