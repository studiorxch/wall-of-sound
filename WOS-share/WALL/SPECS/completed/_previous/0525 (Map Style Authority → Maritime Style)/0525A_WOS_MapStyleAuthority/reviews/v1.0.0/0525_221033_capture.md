# 1. PURPOSE & PRINCIPLES

This specification establishes the authoritative presentation-layer styling system for WOS world rendering. It transforms the system from hardcoded engine rendering routines into a pure, data-authored model where atmospheric palettes, line weights, layering models, and situational styles are determined directly by configuration state records.

### 1.1 Core Tenets
- **Presentation Is Data:** Visual appearance is strictly registry-driven rather than hardcoded inside engine drawings.
- **Runtime Truth Isolation:** Style parameters hold zero write or evaluation authority over `AISRuntime`, telemetry history, or geographic truth. **2D owns truth. 2.5D owns presentation.**
- **Atmosphere Over Utility:** Styling parameters prioritize cinematic readability, low-light observability, and atmospheric persistence over tactical GIS charts or dense dashboard metrics.
- **Passive Interpretation:** The module is permitted to stylize, smooth, or suppress features visually, but it may never fabricate tracking records or distort simulation metrics.

# 2. RUNTIME AUTHORITY BOUNDARIES

### 2.1 Explicit System Ownership
The module asserts exclusive governance over:
- Styling registries, line colors, boundary transparency states, and atmospheric fog factors.
- Maritime sprite scales, lighting halos, and visual interpolation constraints.
- Configuration models across viewplanes (`GEOGRAPHIC_PRIMARY`, `DATA_OVERLAY_ANALYTICS`, `TACTICAL_MINIMAP`).
- Out-of-band hot-reloading configurations for live style editing tools.

### 2.2 Prohibited Modification Scopes
The system has **zero authority** to write or alter parameters inside:
- Navigation vectors, sensor snapshots, or positioning logs in `AISRuntime`.
- Track segment buffers, intensity values, or buffer structures inside `MaritimeWakeAuthority`.
- Target focus loops, ranking variables, or entity tier balances within `MaritimePopulationHierarchy`.
- Visibility tier determinations (`FULL`, `SILHOUETTE`, `MARKER`, `HIDDEN`) resolved by `AtmosphericReadability`.
