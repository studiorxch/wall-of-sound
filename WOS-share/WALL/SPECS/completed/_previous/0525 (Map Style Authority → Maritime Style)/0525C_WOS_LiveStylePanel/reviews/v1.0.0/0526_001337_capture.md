Executive Summary
This specification defines the developer-facing live presentation tuning panel for WOS style registries. It enables visual iteration without runtime mutation, enforces single-writer override governance, validates presentation-only fields, and defers serialization to future specs.

Verdict: ACCEPT as canonical-draft ﾑ authority boundaries clear, single-writer governance correct, validation rules complete, and ready for REVIEW progression to BUILD.

This is a model tooling specification that properly constrains itself to presentation-only edits.

Strengths (Preserve)
1. Core Doctrine: "Live Tooling Is Not Runtime Authority"
"Live tooling may tune appearance. It may not mutate reality."

This is the constitutional foundation for the panel. Preserve verbatim.

2. Single-Writer Override Governance
"Only one active override authority may exist at a time. The panel may not stack multiple competing overrides. It may not create hidden renderer-local mutations."

Enforces 0525A's single-writer rule. Prevents override chaos.

3. Ephemeral by Default
"A value tuned in the LiveStylePanel is temporary until explicitly serialized by a future approved preset serialization path. Panel edits are ephemeral by default."

Prevents accidental production contamination.

4. UI Must Reveal Authority State
"The panel must clearly show whether edits are: inactive, live ephemeral override, serialized preset candidate, invalid, or blocked by governance rule. Hidden live mutations are forbidden."

Transparency requirement prevents silent overrides.

5. Validation Rules with Authority Violation Flag
ts
type StylePanelValidationError = {
  fieldKey: string;
  message: string;
  authorityViolation: boolean;
};
authorityViolation: true distinguishes governance failures from range errors.

6. Explicit Allowed Control Groups
Comprehensive lists for map style layers (water, land, roads, labels, atmosphere, overlays) and maritime style sections (symbolic, lighting, wake, motion, hover, density response). Prevents scope creep.

7. Runtime Field Blocking
"Any field not explicitly listed as allowed must be rejected. The validation error must include authorityViolation: true."

Prevents panel from accidentally exposing runtime authority controls.

8. Forbidden Path Explicit
Approved path:

text
LiveStylePanel ? StyleOverrideDraft ? Validation ? MapStyleAuthority.setSingleLiveOverride() ? MapStyleManifest ? MarineRenderer
Forbidden:

text
LiveStylePanel ? direct renderer mutation
LiveStylePanel ? runtime state mutation
9. Five Editing Modes
INACTIVE, DRAFT, LIVE_OVERRIDE, INVALID, SERIALIZATION_CANDIDATE ﾑ clear state machine.

10. Deferred Serialization
"Actual persistence is deferred to 0525G_WOS_PresentationPresetSerialization_v1.0.0"

Clean separation between live tuning and production persistence.

Fix:

diff
- createdAtMs: Date.now(),

Amendment 6: DEFAULT_DRAFT_EXPIRATION_MS = null ﾑ No Expiration Behavior Defined
Issue: Constant defined as null but no behavior for when/if drafts expire.

Fix: Add clarification:

diff
+ ## Draft Expiration
+ 
+ DEFAULT_DRAFT_EXPIRATION_MS = null means drafts do not auto-expire.
+ 
+ Drafts are cleared when:
+ - Panel is closed (if not saved)
+ - User explicitly clears draft
+ - Another override authority becomes active
+ 
+ Future versions may add auto-expiration; v1.0.0 does not require it.
