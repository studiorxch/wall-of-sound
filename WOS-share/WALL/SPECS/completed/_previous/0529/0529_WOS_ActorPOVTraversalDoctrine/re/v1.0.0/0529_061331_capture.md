I've reviewed the spec. This is a well-structured architectural boundary specification that correctly prioritizes conceptual clarity over feature expansion. Here's my analysis and the requested tasks:

Critical Observations
The Core Insight is Correct
Separating Actor (movement truth) from POV (camera interpretation) resolves the current ambiguity. The flight system currently conflates "plane moving through space" with "camera moving through space" ﾑ this explains why the experience feels unclear.

One Missing Constraint
The spec doesn't explicitly forbid a POV without an Actor. Is a free-floating observer camera allowed? If yes, that becomes a special case where the camera is the actor. Recommend adding:

text
Special case: A camera with no parent actor is a CameraDrone actor.
Task Responses
Task 1 ﾑ Stepped speed control
Doable. Implementation pattern:

js
const SPEED_STEPS = [0.25, 0.5, 1, 2, 5, 10, 20, 40, 80];

// Current index tracking
let speedIndex = 2; // 1x

onSpeedDown() {
  speedIndex = Math.max(0, speedIndex - 1);
  setSpeedMultiplier(SPEED_STEPS[speedIndex]);
}

onSpeedUp() {
  speedIndex = Math.min(SPEED_STEPS.length - 1, speedIndex + 1);
  setSpeedMultiplier(SPEED_STEPS[speedIndex]);
}
UI: two buttons flanking displayed value. No slider.
