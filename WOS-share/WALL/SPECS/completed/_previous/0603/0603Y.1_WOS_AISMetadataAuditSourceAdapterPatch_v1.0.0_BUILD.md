# Purpose

Patch `AISVesselMetadataAudit` so it audits the actual live AIS vessel source.

0603Y currently reads only:

```js
SBE.TruthActorRuntime.listActors()
```

But Wall shows live maritime rendering counts:

```text
rendererAIS: 102
isActive: 51
validationActive: 35
```

while the audit reports:

```text
marineActorCount: 0
```

This proves the audit is looking at the wrong source.

0603Y.1 adds a read-only source adapter so the audit can discover marine actors from whichever live AIS source is actually populated.

---

# Root Cause

Current code:

```js
function _tar() { return SBE.TruthActorRuntime; }

function _marineActors() {
  var tar = _tar();
  if (!tar || typeof tar.listActors !== 'function') return [];
  try {
    return tar.listActors().filter(function (a) {
      return _isMarine(a.actorType);
    });
  } catch (e) {
    return [];
  }
}
```

Problem:

```text
SBE.TruthActorRuntime is absent or empty in Wall
```

but maritime/AIS renderers are active.

Therefore:

```text
audit actor universe = []
live rendered vessel universe = 102
```

# Required Behavior

`AISVesselMetadataAudit.audit()` must discover actors from multiple sources in priority order.

Recommended priority:

```text
1. SBE.TruthActorRuntime.listActors()
2. SBE.AISRuntime read-only vessel accessor
3. SBE.Maritime runtime / validation feed read-only accessor
4. SBE.WorldSpaceVehicleLayer / renderer debug snapshot
5. Empty []
```

Do not stop after an empty source.

Only stop when a source returns usable marine records.

---

# Required Source Adapter

Add:

```js
function _collectMarineActors()
```

or replace `_marineActors()` with a source-adapter version.

Return shape:

```js
{
  source: "truth_actor_runtime" | "ais_runtime" | "maritime_validation_feed" | "world_space_vehicle_layer" | "none",
  actors: [],
  sourcesTried: [
    { source, available, count, marineCount, reason }
  ]
}
```

Then `audit()` should use:

```js
var result = _collectMarineActors();
var actors = result.actors;
```

---

# Actor Normalization

Different sources may not return TruthActorRuntime-shaped actors.

Normalize each source into actor-like records:

```js
{
  actorId,
  actorType: "marine.vessel" | "marine.ferry",
  sourceId: "ais_runtime",
  label,
  name,
  mmsi,
  metadata: {
    shipType,
    ship_type,
    aisShipType,
    type,
    vesselType,
    vesselClass,
    lengthM,
    widthM,
    speedKts,
    headingDeg,
    status
  },
  lng,
  lat
}
```

Normalization must be read-only.

Do not write back to the source record.

Use shallow copies.

# Failure Conditions

This patch fails if:

- audit still reports 0 actors while AISRuntime has active vessels
- audit stops at empty TruthActorRuntime
- audit starts AIS/feed/Drive
- audit mutates AIS records
- audit mutates actor records
- audit mutates assignments
- audit changes bridge settings
- private fields are written
- Studio starts feeds
- Wall Drive breaks
- missing runtime crashes
- malformed vessel record crashes

---

# Implementation Guide

- **Where**: Patch `wall/systems/actors/aisVesselMetadataAudit.js` by replacing `_marineActors()` with a multi-source `_collectMarineActors()` adapter and adding `sources()`; patch `wall/systems/presentation/worldSpaceVehicleDebug.js` to expose `_wos.debug.worldActors.aisMetadataSources()`.
- **What**: Run `node --check wall/systems/actors/aisVesselMetadataAudit.js` and `node --check wall/systems/presentation/worldSpaceVehicleDebug.js`; open Wall with maritime boot active; run `_wos.debug.worldActors.aisMetadataSources()` and `_wos.debug.worldActors.aisMetadataAudit()`.
- **Expect**: The audit selects the populated live AIS/maritime source instead of stopping at empty TruthActorRuntime, reports `marineActorCount > 0` when rendered AIS vessels exist, and does not mutate AIS truth, assignments, bridge settings, feeds, Drive, Wall, Studio, or Mapbox behavior.
