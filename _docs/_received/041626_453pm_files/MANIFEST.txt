═════════════════════════════════════════════════════════════════════════════

✅ P1: DIRECTIONAL EMISSION
   → Particles emit along rotation angle (Math.cos/sin)
   → Speed magnitude calculated from velocity vector
   → Arrow visual indicator follows rotation in real-time

✅ P2: SPAWN MODE TOGGLE
   → "center" mode: spawn at emitter position
   → "edge" mode: spawn 14px offset along direction arrow
   → Dropdown UI control

✅ P3: SILENT MODE
   → Silent checkbox suppresses sound event
   → Visual feedback: dark gray ring (#222) when silent
   → Fully functional, just audio-disabled

✅ P4: QUANTIZE SYNC
   → Snap emission interval to beat grid
   → Formula: gridUnit = (60/BPM)*1000 / division
   → Checkbox + division number (1-64)

✅ P5: SLOW EMISSION RANGE
   → Extended from 100-2000ms to 100-8000ms
   → Supports 1 particle per 8 seconds
   → Perfect for slow rhythmic structures

✅ P6: DIRECTIONAL STABILITY
   → Rotation property maintained per-emitter
   → Direction updates in real-time as slider moves
   → Arrow visual always in sync with actual direction

❓ COMMON QUESTIONS
═════════════════════════════════════════════════════════════════════════════

Q: Will my existing projects break?
A: No. All new properties have safe defaults. Backward compatible 100%.

Q: Do I need to modify engine files (collision.js, physics.js)?
A: No. Only main.js, index.html, styles.css modified. No engine changes.

Q: How much overhead does this add?
A: Less than 0.01%. Two trigonometric operations per emission is negligible.

Q: Can emitters aim in any direction?
A: Yes. Rotation slider spans 0–2π radians (full circle).

Q: Can I quantize without MIDI?
A: Yes. Uses internal transport BPM only. No MIDI dependency.

Q: What's the difference between spawn modes?
A: Center: particles spawn at emitter position
   Edge: particles spawn 14px offset along direction arrow

Q: How do I make slow rhythmic structures?
A: Set Rate to 4000–8000ms. Combine with Quantize for grid-locked beats.

Q: Is the "Silent" mode completely audio-free?
A: Yes. No triggerEvent("ballSpawned") call is made when silent=true.

For more questions, see QUICK_REFERENCE.md "Troubleshooting" section.
