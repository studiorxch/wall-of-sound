
## 1.1 Core Execution Philosophy

To protect core processing bandwidth across the `Runtime Layer` and preserve the absolute integrity of `Spatial Continuity`, the ecosystem implements an explicit **Runtime Gating Framework**.

Because WOS functions as a 24/7 ambient observatory rather than a reactive game loop, systems cannot simply be destroyed or spawned arbitrarily. Doing so fractures atmospheric persistence. When a spatial primitive (e.g., `Vessel`, `Aircraft`, `TransitLine`) or an entire environmental sub-system is no longer actively driving a broadcast channel, its execution must be systematically intercepted at the scheduler level.

The objective of runtime gating is twofold:

1. **Maintain Observability Clarity:** Ensure the system fabric remains visible, traceable, and ready for cinematic interpretation.
    
2. **Zero In-Loop Waste:** Prevent unobserved or inactive elements from burning processing cycles, mutating global states, or polluting the `Event Layer`.
    

---

## 1.2 The Gating Matrix

The `Runtime Layer` enforces five strict gating profiles. A component's assignment to a profile determines its footprint within the deterministic execution loop versus its availability to the interpretation layer.

|**Strategy**|**Runtime Loop Footprint**|**State Memory**|**Event Egress Authority**|**Core WOS Use Case**|
|---|---|---|---|---|
|**Hard Disable**|Dropped completely from scheduler|Fully purged / unallocated|Absolute Zero|Defunct data pipelines, unmapped features, or hard-coded test loops.|
|**Soft Suspend**|Frozen; skipped during tick evaluation|Fully Retained in memory|Paused / Buffered|Active primitives moving out of the broadcast frame but awaiting immediate retention.|
|**Low-Frequency Tick**|Throttled cadence (e.g., sub-sampled at 1Hz)|Fully Active|Throttled / Batch-only|Background telemetry pipelines, long-duration transatlantic dead-reckoning.|
|**Visibility Gating**|100% Simulation Tick rate|Fully Active|Fully Active|Primitives running "hot" behind the scenes to generate background event data, but hidden from the visual camera renderer.|
|**Authority Dormancy**|100% Simulation Tick rate|Fully Active|**Stripped (Read-Only)**|Parallel shadow systems or historical tracking arrays that must read the world state but cannot broadcast back into it.|

## 1.3 The Dormant State Profile

For infrastructure primitives and channels designated as **Dormant**, WOS rejects both raw deletion and hyperactive simulation. Instead, the architecture couples **Soft Suspend** and **Authority Dormancy** into a single, unified runtime gate.

This compound strategy ensures that **environmental continuity is preserved in memory** while the execution cost drops to near-zero.

```
                  [ BroadcastScheduler / GridRuntime Tick ]
                                      │
                                      ▼
                        ┌──────────────────────────┐
                        │    Runtime Gate Check    │
                        └────────────┬─────────────┘
                                     │
                          Is Primitive Dormant? = YES
                                     │
                                     ▼
              ┌──────────────────────────────────────────────┐
              │          COMPOUND GATING ENFORCEMENT          │
              └──────┬────────────────────────────────┬──────┘
                     │                                │
                     ▼                                ▼
          [ GATE 1: Soft Suspend ]       [ GATE 2: Authority Dormancy ]
          • Intercept update loops       • Intercept Event Layer egress
          • Lock state variables         • Enforce Read-Only state access
          • Prevent tick processing      • Sever outbound network mutations
```

### 1.3.1 Execution Gate: Soft Suspend

- **Mechanic:** The `GridRuntime` or `BroadcastScheduler` short-circuits execution before entering the component's update block. The component is bypassed entirely during the active processing frame.
    
- **State Behavior:** The primitive's structural geography, current coordinates, routing telemetry, and historical buffer are securely locked in memory. No destructive teardown or re-initialization routines are permitted. The system is "paused" mid-drift.
    

### 1.3.2 Structural Gate: Authority Dormancy

- **Mechanic:** The system’s egress channels into the `Event Layer` are throttled or decoupled. The component loses all operational clearance to generate events, signal the `Audio Layer`, alter global atmospheric variables, or influence adjacent active primitives.
    
- **State Behavior:** The system exists solely as a deterministic spatial anchor. It can passively read the progression of the world, but it cannot write back to it or trigger emergent music signatures.
    

> 📜 **Doctrine Rule:** A dormant system must remain structurally visible to the architecture to guarantee seamless continuity tracking, but it must remain executionally silent to the processor and behaviorally impotent to the `Event Layer`.

## 1.4 Gated Transitions & Legacy Infrastructure

When a primitive or system architecture is superseded by an improved variant (e.g., migrating from a legacy telemetry parser to a live AIS vessel stream), it transitions away from active runtime scheduling via the **Legacy Pipeline**:

- **Connected Legacy:** Gated via a compound configuration of **Low-Frequency Tick** and **Authority Dormancy**. The legacy component continues to trickle at a heavily reduced cadence strictly to maintain background dependency stability or provide backward-compatible spatial truth for older surfaces.
    
- **Disconnected Legacy:** Gated via **Hard Disable**. The component is completely unmounted from the runtime environment, snapshotted to `wall/persistence`, and wiped from live memory allocation.

## 1.5 Interpretation Layer Rules (UI/UX Philosophy)

When the `Runtime Layer` throws the gate to **Dormant (Soft Suspend + Authority Dormancy)**, the `Interpretation Layer` must translate this shift into cinematic atmosphere rather than a abrupt UI state change.

The presentation must adhere to three UX pillars:

### 1.5.1 Cinematic Decay (Anti-Popping)

- **The Rule:** Primitives must never instantly vanish from the broadcast surface when entering dormancy.
    
- **The Mechanic:** The visual representation (`Vessel` wake, `Vehicle` light trail, or `District` activity glow) must undergo a procedural fade, cinematic drift, or visual decay. The interpretation layer reads the last cached memory state from the **Soft Suspend** buffer and uses inertia to let the asset gracefully settle into the background ambience.
    

### 1.5.2 Low-Light Observability Grammar

- **The Rule:** A dormant system must remain identifiable as a structural anchor without creating visual noise.
    
- **The Mechanic:** Switch the rendering profile of the dormant primitive from an active "hot" signal to a sparse, tactical overlay styling (e.g., matching the aesthetic of harbor radar or public infrastructure surveillance feeds). It drops its active visual emittance and transitions into a muted, static symbolic element embedded in the map's geography.
    

### 1.5.3 Atmospheric Silence over Spectacle

- **The Rule:** Inactive elements must not demand attention.
    
- **The Mechanic:** If a dormant channel or primitive is selected by the `BroadcastScheduler` for a long-duration camera drift, the camera must treat the dormancy as **environmental silence**. The camera pacing slows down, matching the lack of processing ticks with gliding motion, sparse overlay data density, and an emphasis on the surrounding fog or haze.
    

---

### Why this addition completes the UI/UX loop:

- **It honors "Passive Inhabitation":** It treats UI elements like infrastructure. If a train stops sending active updates, a typical game engine deletes it. WOS dims its radar blip, keeps its geographical position visible, and lets the environment swallow it cleanly.
    
- **It standardizes the Overlay Grammar:** It gives your rendering engine (`wall/render/overlays`) a clear mandate: when a system's authority is stripped, its visual representation must switch from "expressive/active" to "symbolic/passive."
